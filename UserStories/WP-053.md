# WP-053: Prompt for reference strategy when renaming a schema

- **ID**: WP-053
- **Description**: As a technical user, when I change a schema name in the schema form, I want the app to ask how to handle existing references, so I can choose a safe migration path.
- **Acceptance criteria**:
  - When editing a schema and changing its name before save, the app opens a confirmation modal.
  - The modal presents exactly two actions:
    - Rename all schema references to the new schema name.
    - Create a new schema and keep existing references unchanged.
  - No schema rename is committed until the user chooses one of the two actions.
  - If the user dismisses/cancels the modal, no rename/create operation is applied.
  - Choosing "Rename all schema references" updates references across all direct and nested usages.
  - After successful rename-all, previous references resolve to the renamed schema without broken references.
