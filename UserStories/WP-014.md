# WP-014: Define and validate scalar query parameters

- **ID**: WP-014
- **Description**: As a technical user, I want to define scalar query parameters and their validation rules, so I can document simple parameters precisely.
- **Acceptance criteria**:
  - Type selector offers: string, number, integer, and boolean.
  - User can create and save a scalar parameter with one of those types.
  - Tree view shows the selected type clearly for each scalar parameter.
  - Scalar parameter types remain stable through edits and persistence.
  - Type can be changed later from the editing panel.
  - Number and integer parameters support minimum and maximum values.
  - String parameters support a regex pattern.
  - All scalar parameters support required, default value, and description.
  - Validation prevents a minimum greater than a maximum.
  - Validation rejects invalid regex patterns.
  - Validation rejects default values that do not match the selected type.
  - Validation messages are shown inline and are understandable to a technical user.
