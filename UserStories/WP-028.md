# WP-028: Add an apiKey security requirement to an operation

- **ID**: WP-028
- **Description**: As a technical user, I want to add an `apiKey` security requirement to an operation by defining its name and location, so I can document key-based authentication for that endpoint.
- **Acceptance criteria**:
  - Selecting "Add security" opens a form (inline or side panel consistent with query parameter edit pattern).
  - The form includes a "Type" selector with options: `apiKey` and `http`.
  - When `apiKey` is selected, the form shows:
    - **Name**: free-text field (used as the scheme identifier, e.g. `ApiKeyAuth`).
    - **In**: selector with values `header`, `query`, `cookie`.
    - **Key name**: free-text field for the actual parameter name (e.g. `X-API-Key`).
  - All three fields are required; the form cannot be saved with empty values.
  - Saving adds the security requirement to the operation's security array.
  - The scheme definition is stored under `components/securitySchemes` using the provided name.
  - If a scheme with the same name already exists on a different operation, the form shows a warning but still allows saving.
  - The new entry appears immediately in the security panel list.
