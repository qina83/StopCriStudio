# WP-045: Persist and load operation responses

- **ID**: WP-045
- **Description**: As a technical user, I want operation responses to persist and load correctly from local storage and imported files, so I can continue editing without losing response definitions.
- **Acceptance criteria**:
  - Any create, edit, or delete action in the Responses section is autosaved to local storage.
  - Reloading the page restores all operation responses exactly as previously saved.
  - Loading a specification file parses responses for all operations, including multiple status codes and media types.
  - Loaded responses preserve both inline schemas and `$ref` models.
  - Invalid or unsupported response fragments from imported files are preserved in specification content and shown as read-only where editing is not supported.
  - Response data remains correctly scoped to each path and operation after load.
