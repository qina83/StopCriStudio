# WP-002.4: User edits path with operation management form

- **ID**: WP-002.4
- **Description**: As a technical user, I want to edit a selected path with a dedicated form showing all available HTTP operations and the ability to add or remove operations, so I can manage the API endpoints comprehensively in one view.
- **Acceptance criteria**:
  - When user selects a path from the list, a form appears in the main panel
  - Form displays the path name/details at the top
  - Form displays all available HTTP methods as interactive buttons (GET, POST, PUT, DELETE, PATCH, HEAD, OPTIONS)
  - Available HTTP method buttons that are NOT yet added to the path are clickable and enabled
  - HTTP method buttons that already exist as operations are marked as "already added"
  - Each HTTP method button displays the method name clearly and is color-coded by method type
  - When user clicks an HTTP method button (not yet added), a confirmation modal appears
  - Confirmation modal displays the method name (e.g., "Add GET operation?") and Yes/No buttons
  - Clicking "Yes" adds the operation to the path
  - Clicking "No" closes the modal without adding the operation
  - After operation is added, the form updates and the HTTP button is marked as added
  - When user clicks on an existing operation in the form, that operation becomes selected
  - Selected operation displays prominently with the operation name in large text (e.g., "POST")
  - A "Delete" button appears next to the selected operation name
  - Selected operation details (summary, description) display below the large operation name
  - Clicking the "Delete" button shows a confirmation modal with warning and Yes/No buttons
  - Clicking "Yes" in delete modal removes the operation from the path
  - After deletion, the form updates and the operation's HTTP button becomes available again
  - The large operation name display clears after deletion
  - User can add and remove multiple operations from the same path without page refresh
  - All changes are automatically saved to local storage
  - Selecting a different path updates the form to show that path's operations
  - The form remains visible until user navigates away or deselects the path
