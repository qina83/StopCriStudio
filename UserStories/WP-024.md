# WP-024: Select media type for request body content

- **ID**: WP-024
- **Description**: As a technical user, I want to select one media type for the request body content, so that the specification correctly identifies the format consumers must use when sending requests.
- **Acceptance criteria**:
  - The body panel includes a media type selector that appears before the parameters section.
  - The selector offers at least the following options: `application/json`, `application/x-www-form-urlencoded`, `multipart/form-data`, `text/plain`, `application/octet-stream`.
  - The user can also type a custom media type value not present in the predefined list.
  - Only one media type is allowed per operation body; selecting a new media type replaces the previous one.
  - Changing the media type preserves existing body parameters if the schema structure is compatible; a confirmation prompt is shown if the change would clear existing parameters.
  - The selected media type defaults to `application/json` when the body section is first opened for an operation.
  - The selected media type is persisted to local storage on change.
  - An empty media type value is not allowed; the selector must always have a valid value.

## User interaction flow

*The user opens the Body panel for a PUT operation. The media type defaults to `application/json`. The user changes it to `multipart/form-data`. A confirmation is shown if parameters already exist. After confirming, the media type updates and is auto-saved.*
