# WP-043: Define response model as reusable schema reference

- **ID**: WP-043
- **Description**: As a technical user, I want to set a response model using a reusable schema reference, so I can keep response payloads consistent across operations and reduce duplication.
- **Acceptance criteria**:
  - For any response media type entry, the user can choose "Schema reference" as model source.
  - The schema selector lists available entries from `components.schemas`.
  - Selecting a schema stores the model as `$ref: "#/components/schemas/{schemaName}"` at the response schema location.
  - Switching from inline schema to schema reference requires confirmation when inline content would be replaced.
  - Switching from schema reference to inline schema requires confirmation when the `$ref` would be replaced.
  - If a referenced schema is renamed elsewhere, response `$ref` values are updated automatically.
  - If a referenced schema is deleted, impacted responses are clearly flagged until a new model is selected.
