# WP-046: Export operation responses to OpenAPI YAML/JSON

- **ID**: WP-046
- **Description**: As a technical user, I want all operation responses to be exported in valid OpenAPI format, so the generated YAML/JSON can be consumed by tools and clients without manual fixes.
- **Acceptance criteria**:
  - Export includes each operation's `responses` object with all defined status codes.
  - For each status code, export includes all defined media types under `content`.
  - Inline response models export as valid schema objects at the correct response schema location.
  - Reusable response models export as `$ref` values under the correct response schema location.
  - Response descriptions are exported when provided.
  - Operations with no configured responses still export at least one valid OpenAPI response entry (for example, default initialized response) according to editor behavior.
  - Exported output validates against OpenAPI 3.0/3.1 response object structure for covered features.
