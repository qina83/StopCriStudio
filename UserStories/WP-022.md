# WP-022: Access request body panel for eligible operations

- **ID**: WP-022
- **Description**: As a technical user, I want the request body section to be available only for PUT, POST, and PATCH operations, so that the interface reflects valid OpenAPI semantics and avoids confusion for methods that do not support a body.
- **Acceptance criteria**:
  - A "Body" tab or section is visible in the operation editor when the HTTP method is PUT, POST, or PATCH.
  - The "Body" tab or section is hidden and not accessible when the HTTP method is GET, DELETE, HEAD, OPTIONS, or TRACE.
  - Switching an operation's method from a body-eligible method to a non-eligible one hides the body section and does not retain body data in the UI.
  - Switching back to a body-eligible method restores previously entered body data if it was already persisted.
  - The body section placement and styling follow the same visual pattern as the Query Parameters panel.
  - No body-related data is exported for non-eligible methods even if residual data exists in storage.

## User interaction flow

*The user selects or views a POST operation. The "Body" tab appears alongside other operation tabs. When the user switches to a GET operation, the "Body" tab disappears. When the user switches back to POST, the body data is restored.*
