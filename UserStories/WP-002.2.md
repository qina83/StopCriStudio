# WP-002.2: User selects created path and accesses operations

- **ID**: WP-002.2
- **Description**: As a technical user, I want the newly created path to be automatically selected and the operations section to appear in the path form, so I can immediately start adding operations to the path without additional navigation steps.
- **Acceptance criteria**:
  - When user creates a new path and confirms it, the path is automatically selected in the paths list
  - The newly created path is highlighted/visually indicated as selected in the paths list
  - The path form remains visible in the main panel after creation
  - The operations section appears in the path form for the selected path
  - Operations section displays an "Add Operation" button (or similar control) to create HTTP operations (GET, POST, PUT, DELETE, etc.)
  - User can immediately start adding operations to the newly created path without additional clicks
  - The path form displays the path details (e.g., path URL) along with the operations section
  - If user clicks on a different path in the list, that path becomes selected and its operations appear in the form
  - The selected path state persists when toggling between form view and list view
  - Operations management is fully functional for the newly created path
