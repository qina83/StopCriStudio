# WP-055: Create reusable schema from inline request body structure

- **ID**: WP-055
- **Description**: As a technical user, I want to create a reusable schema from an inline request body structure, so I can avoid duplication and keep my API models consistent.
- **Acceptance criteria**:
  - When an operation request body is defined with an editable inline object structure, the UI exposes an action to create a schema from that inline structure.
  - Triggering the action asks for a target schema name before any change is persisted.
  - If the schema name is valid and does not already exist, the app creates a new entry in `components.schemas` using the current inline request body object structure.
  - After successful creation, the operation request body references the newly created schema via `$ref` instead of keeping the inline object structure.
  - Request body metadata not part of the schema model (for example media type, request-body required flag, and request-body description) remains unchanged.
  - If the user cancels the action, no schema is created and no request body content is modified.
