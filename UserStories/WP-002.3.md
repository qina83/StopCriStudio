# WP-002.3: User manages operations in path

- **ID**: WP-002.3
- **Description**: As a technical user, I want to create and delete operations within a selected path with confirmation modals, so I can effectively manage the HTTP methods and endpoints for my API specification.
- **Acceptance criteria**:
  - When user clicks "Add Operation" button under a selected path, a confirmation modal appears
  - Modal displays a clear title: "Add Operation" or similar
  - Modal includes a dropdown or selector to choose HTTP method (GET, POST, PUT, DELETE, PATCH, etc.)
  - Modal includes a "Confirm" and "Cancel" button
  - When user clicks "Confirm" in the operation creation modal, the operation is added to the selected path
  - After successful creation, the confirmation modal closes automatically
  - The operations list for the selected path is updated to display the newly created operation
  - When user selects an operation from the operations list, the operation is visually highlighted/selected
  - A "Delete" button becomes visible when an operation is selected
  - "Delete" button is only visible when an operation is actively selected
  - When user clicks the "Delete" button for a selected operation, a confirmation modal appears
  - Delete confirmation modal displays a clear warning message (e.g., "Are you sure you want to delete this operation?")
  - Delete confirmation modal includes "Confirm Delete" and "Cancel" buttons
  - When user clicks "Confirm Delete" in the delete modal, the operation is removed from the path
  - After successful deletion, the delete confirmation modal closes automatically
  - The operations list for the path is updated to reflect the removal
  - The "Delete" button becomes hidden after the operation is deleted
  - User can create and delete multiple operations without issues
  - All changes are automatically saved to local storage
  - Operations are displayed in a clear list format under the selected path
  - Creating or deleting operations does not interfere with other path or specification data
