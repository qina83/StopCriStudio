# WP-052: Show schema usage list in schema form

- **ID**: WP-052
- **Description**: As a technical user, I want to see where a schema is used while editing it, so I can understand impact before making changes.
- **Acceptance criteria**:
  - The schema form shows a "Used in" section at the bottom when editing an existing schema.
  - The "Used in" section lists all operations that reference the schema, including nested and indirect usages.
  - Nested usages include references found inside composed schemas and nested structures (for example arrays, object properties, and composition constructs).
  - Each usage item displays only: HTTP method + path (for example: `GET /users/{id}`).
  - Duplicate usages for the same method + path are collapsed into a single list item.
  - If the schema has no usages, the section shows a clear empty state message.
  - The usage list updates after edits that affect references without requiring a full page reload.
