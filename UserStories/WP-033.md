# WP-033: Load operation security from file or local storage

- **ID**: WP-033
- **Description**: As a technical user, I want security requirements to be correctly parsed and displayed when I load a specification from a file or from local storage, so I can continue working on previously defined security without data loss.
- **Acceptance criteria**:
  - When loading a specification from a file, `security` arrays on operations are parsed and displayed in each operation's security panel.
  - When loading a specification from local storage, the same parsing and display applies.
  - Each loaded security entry is matched to its scheme definition in `components/securitySchemes` and displays the correct type (`apiKey` or `http`) and all relevant fields.
  - Scheme types other than `apiKey` and `http` (e.g. `oauth2`, `openIdConnect`) are preserved in the data but displayed as read-only entries with a clear label indicating they are unsupported for editing.
  - If a `security` entry references a scheme name not present in `components/securitySchemes`, it is still shown in the list with a visual warning indicating the missing definition.
  - All loaded security data is immediately available for editing via the standard add/edit/delete actions.
  - Loading does not overwrite security data already in local storage unless the user explicitly chooses to load and replace.
