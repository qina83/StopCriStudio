# WP-021: Use the query parameter tree editor

- **ID**: WP-021
- **Description**: As a technical user, I want a tree-based editor for query parameters, so I can understand structure, edit quickly, and stay aware of the current operation.
- **Acceptance criteria**:
  - The editor clearly shows the current operation context using HTTP method and path.
  - Query parameters are rendered as a tree with root-level parameters visible by default.
  - Primitive parameters appear as leaf nodes.
  - Object parameters appear as expandable and collapsible nodes.
  - Array parameters show their element type in the tree.
  - Each visible node shows name, type, and whether the field is required.
  - Tree indentation makes nesting depth easy to understand.
  - Node-level actions expose edit and delete without forcing the user to leave the tree.
  - Editing opens a side panel or modal while preserving visibility of the tree.
  - The tree supports enough parameters and nesting depth to remain usable for complex operations.
