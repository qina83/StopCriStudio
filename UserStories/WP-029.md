# WP-029: Add an http security requirement to an operation

- **ID**: WP-029
- **Description**: As a technical user, I want to add an `http` security requirement to an operation by selecting the scheme type, so I can document Basic or Bearer authentication for that endpoint.
- **Acceptance criteria**:
  - When `http` is selected as the type, the form shows:
    - **Name**: free-text field (used as the scheme identifier, e.g. `BearerAuth`).
    - **Scheme**: selector with values `basic` and `bearer`.
    - **Bearer format**: optional free-text field (e.g. `JWT`), visible only when `bearer` is selected.
  - Name and Scheme are required; bearer format is optional.
  - The form cannot be saved without name and scheme values.
  - Saving adds the security requirement to the operation's security array.
  - The scheme definition is stored under `components/securitySchemes` using the provided name.
  - The bearer format is included in the scheme definition only when a non-empty value is set.
  - The new entry appears immediately in the security panel list.
