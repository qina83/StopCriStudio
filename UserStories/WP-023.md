# WP-023: Configure request body metadata

- **ID**: WP-023
- **Description**: As a technical user, I want to set a description and a required flag on the request body, so that the generated OpenAPI specification accurately documents whether callers must provide a body and what it represents.
- **Acceptance criteria**:
  - The body panel contains a "Description" text field that is optional and maps to the OpenAPI `requestBody.description` field.
  - The body panel contains a "Required" toggle or checkbox that maps to the OpenAPI `requestBody.required` field (defaults to `false`).
  - Both fields are clearly labelled and follow the visual style of equivalent fields in other panels (e.g., Info panel, path parameter form).
  - Changing either field updates the in-memory state immediately.
  - Both fields are persisted to local storage on change (auto-save, no manual save needed).
  - Description accepts plain text; no markdown rendering is required at this stage.
  - The required flag value is always exported as an explicit boolean (`true` or `false`).

## User interaction flow

*The user opens the Body panel for a POST operation, types an optional description, and toggles the Required switch to true. The values are auto-saved and appear correctly in the exported YAML.*
