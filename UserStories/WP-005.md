# WP-005: User edits path name inline

- **ID**: WP-005
- **Description**: As a technical user, I want to edit the name of an existing path directly from the path form header, so I can rename API endpoints without losing their associated operations.
- **Acceptance criteria**:
  - When user selects a path from the list, the path form displays with the path name shown in the header
  - An edit icon (pencil icon) appears next to the path name in the form header when a path is selected
  - Edit icon is displayed to the right of the path name text
  - Edit icon is clearly visible and has appropriate hover styling (color change, scale, or background highlight)
  - When user clicks the edit icon, the path name becomes an inline editable text field
  - The text field is activated with the current path name pre-filled and selected for editing
  - Text field has the same visual styling as the path creation form for consistency
  - User can type freely to modify the path name in the text field
  - Text field displays a placeholder or clear indication that it's in edit mode
  - User can press Enter to confirm the path name change
  - User can press Escape to cancel editing and revert to the original path name
  - A confirmation button (e.g., checkmark icon or "Save" button) appears next to the text field as an alternative to pressing Enter
  - A cancel button (e.g., X icon or "Cancel" button) appears next to the confirmation button
  - Clicking the confirmation button saves the new path name
  - Clicking the cancel button reverts to the original path name
  - When user renames a path, all associated HTTP operations automatically transfer to the new path name
  - All operation details (summaries, descriptions, parameters, etc.) are preserved during the rename
  - The path list in the sidebar updates to show the new path name
  - The selected path remains selected after renaming (user sees the updated name highlighted)
  - Path name validation prevents duplicate path names - if a path name already exists, an error message appears
  - Error message clearly explains that the path name already exists and requests a different name
  - User can correct the duplicate name and retry saving
  - Path names with invalid characters trigger a validation error with helpful guidance
  - Leading and trailing whitespace is automatically trimmed from the path name
  - Path name cannot be empty - attempting to save an empty name shows validation error
  - After successful rename, the text field returns to display mode showing the new path name
  - All changes are automatically saved to local storage
  - User can edit the same or different paths multiple times without issues
  - Edit mode closes automatically after save or cancel without affecting the rest of the interface
  - Selecting a different path while in edit mode either cancels the edit or prompts user to save/discard changes
  - The form header appearance remains consistent with the existing path form design
