# WP-034: Create a reusable schema

- **ID**: WP-034
- **Description**: As a technical user, I want to create a named, reusable schema in the Schemas panel using the same tree-based field editor I use for request bodies, so I can define data models once and reference them across my specification.
- **Acceptance criteria**:
  - The Schemas panel displays an "Add Schema" button.
  - Clicking "Add Schema" opens an inline form (or modal) that requires a schema name (e.g. `UserResponse`).
  - The schema name must be a non-empty string containing only alphanumeric characters, hyphens, and underscores; a clear validation error is shown otherwise.
  - Once named, the user enters a tree-based field editor identical in behaviour to the request body editor: fields can be added with a name, type (string, number, integer, boolean, object, array), required flag, and description.
  - Object fields can have nested properties; array fields can have a defined item type with optional nested properties when `itemType` is `object`.
  - The schema is always created with `type: object` at the top level; the user cannot change the top-level type.
  - Saving the form adds the schema to `components/schemas` in the specification content.
  - The new schema appears immediately in the Schemas panel list without a page reload.
  - If a schema with the same name already exists, a validation error is shown and the save is blocked.
