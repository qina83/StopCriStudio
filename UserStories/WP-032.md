# WP-032: Export operation-level security to YAML

- **ID**: WP-032
- **Description**: As a technical user, I want operation security requirements to be included in the exported OpenAPI YAML, so the generated specification is complete and valid for consumers and tooling.
- **Acceptance criteria**:
  - The exported YAML includes a `security` array under each operation that has at least one requirement defined.
  - Each entry in the `security` array follows the OpenAPI Security Requirement Object format: a map of scheme name to an empty array of scopes (`[]`).
  - Operations with no security requirements defined do not include a `security` key in the export.
  - The exported YAML includes a `components.securitySchemes` map containing all scheme definitions referenced by any operation.
  - `apiKey` schemes export with `type: apiKey`, `in`, and `name` fields.
  - `http` schemes export with `type: http` and `scheme` fields; `bearerFormat` is included only when set.
  - Generated YAML is valid for OpenAPI 3.0 and 3.1.
