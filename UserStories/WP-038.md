# WP-038: Persist and load schemas from local storage and file

- **ID**: WP-038
- **Description**: As a technical user, I want schemas defined in `components/schemas` to be saved to local storage automatically and correctly restored when I reload the application or load a specification from a file, so I never lose my data models.
- **Acceptance criteria**:
  - Whenever a schema is created, edited, or deleted, the change is persisted to local storage as part of the full specification content.
  - When the application reloads and restores a specification from local storage, all schemas in `components/schemas` are parsed and displayed in the Schemas panel.
  - When loading a specification from a file, schemas present in `components/schemas` are parsed and displayed correctly in the Schemas panel.
  - Each loaded schema is rendered in the same tree-based view used during creation, showing all fields and their types at every nesting level.
  - If a loaded schema has an unsupported structure (e.g. `allOf`, `oneOf`, `anyOf`, or a non-object top-level type), it is displayed as a read-only raw entry with a clear label indicating it cannot be edited in the visual editor.
  - `$ref` fields within request bodies that reference a schema in `components/schemas` continue to display the correct read-only preview after loading.
  - Loading does not overwrite existing schema data in local storage unless the user explicitly chooses to load and replace the full specification.
