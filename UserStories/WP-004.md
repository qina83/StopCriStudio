# WP-004: User exports specification as YAML

- **ID**: WP-004
- **Description**: As a technical user, I want to export my OpenAPI specification as a YAML file, so I can use it in other systems and share it with team members.
- **Acceptance criteria**:
  - Export button is visible in the header, top right corner
  - Export button is labeled "Export" and clearly visible
  - Clicking Export button triggers YAML file download
  - Downloaded file is named using the specification name defined in the info panel (e.g., "my-api.yaml")
  - If specification name is empty or invalid, filename defaults to "openapi.yaml"
  - YAML export contains the complete specification as-is, without validation
  - Downloaded file is in valid YAML format
  - Export works from any editor screen (specification info, paths, schemas, etc.)
  - Export button remains accessible and functional throughout the editing workflow
  - File downloads to user's default download location using browser's download mechanism
