# WP-031: Persist operation security to local storage

- **ID**: WP-031
- **Description**: As a technical user, I want security requirements to auto-save to local storage after every change, so I never lose my work between sessions.
- **Acceptance criteria**:
  - Every add, edit, or delete action on security requirements triggers immediate persistence to local storage.
  - Both the operation-level `security` array and the `components/securitySchemes` definitions are persisted together.
  - Reloading the page restores all security requirements for all operations.
  - No manual save action is required.
  - Storage quota errors, if encountered, surface a user-friendly error message.
