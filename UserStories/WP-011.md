# WP-011: Manage query parameters for an operation

- **ID**: WP-011
- **Description**: As a technical user, I want to add, edit, and remove query parameters for a specific API operation, so I can keep each endpoint's query contract accurate.
- **Acceptance criteria**:
  - User can navigate to a specific operation identified by HTTP method and path.
  - A "Query parameters" section is visible within the operation editor.
  - User can click "Add parameter" to create a new query parameter.
  - Parameter creation includes at least: name and type.
  - New parameters are added to the current operation only.
  - Existing parameters can be edited from the same query parameters experience.
  - Existing parameters can be removed without a confirmation dialog.
  - After add, edit, or remove, the visible parameter list updates immediately.
  - Empty states show a clear message instead of a blank area.
  - Stored query parameters map to the operation's parameters array in the OpenAPI structure.
