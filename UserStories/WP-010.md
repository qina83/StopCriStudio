# WP-010: User edits API information from header

- **ID**: WP-010
- **Description**: As a user, I want to edit API information directly from the header using an edit icon, so I have quicker access to this functionality without navigating through the sidebar.
- **Acceptance criteria**:
  - The "Info" button is removed from the sidebar navigation
  - An edit icon (pencil or similar) appears in the header next to the API specification title
  - The edit icon is positioned to the right of the specification name and version
  - When user clicks the edit icon, the API Information form is opened (same form that currently opens when clicking "Info" from sidebar)
  - The edit icon has a tooltip/aria-label indicating its purpose (e.g., "Edit API Information")
  - The edit icon has hover and focus states for better UX
  - The clicked edit icon visually indicates the active state (similar to how "Info" was highlighted in sidebar)
  - The information form displays all the same fields as before (no functionality changes)
  - User can close the information form and return to viewing other sections
  - When the information form is open, the edit icon remains highlighted/active
  - The header layout remains clean and aligned with existing design patterns
  - The edit functionality works consistently across all page sizes and screen resolutions

## User interaction flow

*The user sees the specification title and version in the header, along with an edit icon. Clicking the edit icon reveals the API Information form. Once closed, the icon returns to normal state.*
