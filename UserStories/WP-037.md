# WP-037: Reference a schema ($ref) in a request body object field

- **ID**: WP-037
- **Description**: As a technical user, when I define a request body field of type `object`, I want the option to reference an existing schema from `components/schemas` instead of defining the properties inline, so I can keep my specification DRY and consistent.
- **Acceptance criteria**:
  - In the request body tree editor, when the user adds or edits a field with type `object`, a toggle or mode selector is shown: "Define inline" (default) vs "Use schema reference ($ref)".
  - Selecting "Use schema reference ($ref)" replaces the inline property tree with a dropdown listing all schemas currently defined in `components/schemas`.
  - The dropdown is searchable if there are more than five schemas.
  - If no schemas are defined yet, the option is disabled with a tooltip explaining that schemas must be created first in the Schemas panel.
  - Selecting a schema from the dropdown sets the field value to `{ "$ref": "#/components/schemas/{schemaName}" }` in the specification content.
  - After selection, the referenced schema's fields are displayed read-only beneath the field row (collapsed by default, expandable).
  - The user cannot edit the referenced schema's fields from within the request body editor; a link or label directs them to the Schemas panel to make changes.
  - Switching back to "Define inline" clears the `$ref` and reverts to an empty inline property tree (with a confirmation if a schema was already selected).
  - The `$ref` value is correctly serialised into the exported OpenAPI YAML/JSON.
