# WP-041: Add operation responses with multi status codes and media types

- **ID**: WP-041
- **Description**: As a technical user, I want to add multiple responses to an operation using different HTTP status codes and media types, so I can model realistic success and error outcomes.
- **Acceptance criteria**:
  - The user can add a response entry by selecting or typing a valid HTTP status code (for example, `200`, `201`, `400`, `404`, `500`) or `default`.
  - The user can define one or more media types per status code (for example, `application/json`, `application/xml`, `text/plain`).
  - Adding a second media type under the same status code does not overwrite existing media types.
  - Adding a new status code creates a separate response node under the same operation.
  - If the same status code and media type combination already exists, save is blocked with a clear validation message.
  - The response list updates immediately after save.
  - All created responses are scoped to the current operation only.
