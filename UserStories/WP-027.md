# WP-027: Access the security panel for an operation

- **ID**: WP-027
- **Description**: As a technical user, I want a dedicated "Security" section within the operation editor, so I can view and manage security requirements for each operation in a consistent way.
- **Acceptance criteria**:
  - A "Security" section is visible within the operation editor for all HTTP methods.
  - The section is placed and styled consistently with the Query Parameters panel.
  - When no security requirements are defined, an empty state message is shown instead of a blank area.
  - The section displays a list of all security requirements already defined for the operation.
  - Each row shows the scheme name and its type (`apiKey` or `http`).
  - An "Add security" action is available to create a new security requirement.
