# WP-040: Access operation response model panel

- **ID**: WP-040
- **Description**: As a technical user, I want a dedicated Responses section in the operation editor, so I can define and manage all response contracts for the selected operation in one place.
- **Acceptance criteria**:
  - A "Responses" section is visible for every selected operation regardless of HTTP method.
  - The section header clearly shows the active operation context (HTTP method + path).
  - The section shows an empty state when no responses are defined.
  - The empty state includes a primary action to add the first response.
  - If responses already exist, they are listed in a structured way by status code and media type.
  - Switching to a different operation refreshes the section and shows only that operation's responses.
