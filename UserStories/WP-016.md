# WP-016: Define object-type query parameters

- **ID**: WP-016
- **Description**: As a technical user, I want to define object-type query parameters with nested properties, so I can model complex query structures.
- **Acceptance criteria**:
  - Type selector includes an object option.
  - When a parameter is set to object, it appears as an expandable node in the tree.
  - Object nodes provide an "Add property" action.
  - New properties require at least a name and type.
  - Properties can be edited and removed after creation.
  - Property names must be unique within the same object.
  - Object properties can use any supported type, including string, number, integer, boolean, object, and array.
  - Object properties can themselves contain nested objects recursively.
  - The hierarchy is visually clear in the tree through indentation and expand or collapse behavior.
  - Deeply nested object structures remain editable and export correctly to YAML.
