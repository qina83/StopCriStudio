# WP-009: User filters paths and schemas in sidebar

- **ID**: WP-009
- **Description**: As a user working with large API specifications, I want to filter the paths and schemas displayed in the sidebar using a search input field, so I can quickly find the specific endpoints and data models I need to edit.
- **Acceptance criteria**:
  - A search input field appears at the top of the sidebar, above the Paths and Schemas accordions
  - The input field displays a clear placeholder text (e.g., "Filter paths & schemas...")
  - The input field has a clear button (X icon) on the right side
  - When user types in the search field, the Paths and Schemas lists are filtered in real-time to show only matching items
  - Filter matching is case-insensitive
  - Filter matching works for partial text (substring matching)
  - Filter is applied with a debounce delay of 300ms to improve performance
  - While debounce delay is active, the previous filter results continue to display
  - When user clicks the X button, the search input field is cleared and the filter is reset
  - All paths and schemas are displayed again after clearing the filter
  - The filter state is not persisted to local storage (clears on page refresh)
  - Filtered paths still display their child operations under the filtered path item
  - If no paths or schemas match the filter, an empty state message appears (e.g., "No matches found")
  - The sidebar layout remains consistent while filtering items

## Filter interaction flow

*The search input field is positioned prominently at the top of the sidebar. As the user types, matching paths and schemas are highlighted or displayed. The clear button allows quick reset of the filter.*
