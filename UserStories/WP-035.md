# WP-035: Edit an existing schema

- **ID**: WP-035
- **Description**: As a technical user, I want to open and edit any schema I have previously defined in the Schemas panel, so I can evolve my data models as my API changes.
- **Acceptance criteria**:
  - Each schema in the Schemas panel list has an "Edit" action (button or clickable row).
  - Clicking "Edit" opens the same tree-based editor pre-populated with the schema's existing fields and structure.
  - The user can rename the schema; renaming validates against the same rules as creation (non-empty, alphanumeric/hyphens/underscores, unique).
  - The user can add, modify, or remove any field at any nesting level using the same interactions available during creation.
  - Saving the changes updates `components/schemas[schemaName]` in the specification content immediately.
  - If the schema was renamed, the old key is removed and the new key is created.
  - When a schema is renamed, all `$ref` values in the specification content that reference the old name (`#/components/schemas/{oldName}`) are automatically updated to the new name (`#/components/schemas/{newName}`).
  - The update applies across all operations' request bodies at every nesting level.
  - A summary message informs the user how many `$ref` occurrences were updated as a result of the rename.
  - Cancelling the editor discards all unsaved changes and returns the user to the Schemas panel list.
