# WP-039: Export schemas to OpenAPI YAML/JSON

- **ID**: WP-039
- **Description**: As a technical user, I want all schemas I have defined to be included in the exported OpenAPI YAML/JSON file, so the output is a complete and valid specification that consumers and tooling can use without modification.
- **Acceptance criteria**:
  - The exported file includes a `components.schemas` map containing every schema defined in the Schemas panel.
  - Each schema is serialised as a valid OpenAPI Schema Object under its exact name key (e.g. `components.schemas.UserResponse`).
  - All field names, types, required arrays, descriptions, and nesting levels are preserved exactly as defined in the editor.
  - Array fields with `itemType: object` export their nested properties correctly under `items.properties`.
  - If no schemas have been defined, `components.schemas` is omitted from the export entirely (no empty object).
  - Request body fields that use a `$ref` to a schema export as `{ "$ref": "#/components/schemas/{schemaName}" }` at the correct position in the operation's `requestBody.content` schema.
  - The exported YAML/JSON passes OpenAPI 3.0 validation for `components/schemas` structure.
  - Schemas with unsupported structures that were loaded read-only (e.g. `allOf`, `oneOf`) are preserved verbatim in the export without modification.
