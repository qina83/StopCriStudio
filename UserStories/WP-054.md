# WP-054: Create schema copy on rename choice and block name conflicts

- **ID**: WP-054
- **Description**: As a technical user, if I choose to create a new schema after changing a schema name, I want the new schema to be cloned from the edited schema and blocked on name conflicts, so no existing schema is overwritten.
- **Acceptance criteria**:
  - If the user selects "Create a new schema and keep existing references unchanged," the app creates a new schema entry using the new name.
  - The new schema is a clone of the current edited schema content at save time.
  - Existing references remain pointed to the original schema name and are not modified.
  - If the target new schema name already exists, creation is blocked and a clear validation message is shown.
  - When name conflict occurs, no partial schema create or reference update is persisted.
  - On successful creation, both original and new schema are available in schema lists and editor flows.
