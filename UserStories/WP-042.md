# WP-042: Define response model as inline schema

- **ID**: WP-042
- **Description**: As a technical user, I want to define a response model inline for a specific status code and media type, so I can quickly model operation-specific payloads without creating reusable schemas.
- **Acceptance criteria**:
  - For any response media type entry, the user can choose "Inline schema" as model source.
  - The inline model editor uses the same tree-based interaction pattern as the request body editor.
  - The user can add, edit, and remove fields at multiple nesting levels.
  - Field metadata (type, required, description, nested properties, array items) is supported using the same rules as request body schema editing.
  - Saving stores the schema under the correct OpenAPI location: `paths.{path}.{method}.responses.{statusCode}.content.{mediaType}.schema`.
  - Unsaved inline changes are discarded when the user cancels editing.
  - Inline schema changes are reflected immediately in the response summary list.
