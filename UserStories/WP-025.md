# WP-025: Manage body schema parameters

- **ID**: WP-025
- **Description**: As a technical user, I want to add, edit, and remove properties in the request body schema using the same tree-based editor pattern as query parameters, so I can define the body structure intuitively without context-switching.
- **Acceptance criteria**:
  - Below the media type selector, a tree editor displays the inline schema properties of the request body.
  - The user can add a new root-level property using an "Add property" action.
  - Each property has at minimum: name, type (string, number, integer, boolean, object, array), required flag, and optional description.
  - Primitive properties (string, number, integer, boolean) appear as leaf nodes in the tree.
  - Object properties appear as expandable/collapsible nodes; nested properties can be added within them.
  - Array properties show their item type; item type can be set to any primitive or object.
  - Each node exposes inline edit and delete actions without leaving the tree view.
  - Editing a node opens a side panel or inline form consistent with the query parameter edit pattern.
  - Deleting a node that has children asks for confirmation before removing the node and all its descendants.
  - Property names must be unique within the same parent object; duplicate names at the same level are not allowed.
  - An empty body (no properties defined) is valid and results in an empty schema object in the export.
  - All add/edit/delete actions trigger auto-save to local storage immediately.

## User interaction flow

*The user adds a root-level property named `username` of type `string`, marks it as required, then adds a nested object `address` with child properties `street` and `city`. Each change is auto-saved. The user deletes `address` and confirms removal of it and its children.*
