# WP-030: Edit and remove security requirements on an operation

- **ID**: WP-030
- **Description**: As a technical user, I want to edit or remove existing security requirements on an operation, so I can keep the operation's security contract accurate as the API evolves.
- **Acceptance criteria**:
  - Each security requirement row in the list exposes inline edit and delete actions.
  - Clicking edit opens the same form used during creation, pre-populated with the current values.
  - All fields can be updated; saving replaces the previous values for that requirement.
  - Renaming the scheme identifier updates both the operation's `security` array reference and the `components/securitySchemes` key.
  - Clicking delete removes the security requirement from the operation immediately, without a confirmation dialog.
  - If the deleted scheme is not referenced by any other operation, its definition is also removed from `components/securitySchemes`.
  - If the deleted scheme is still referenced by other operations, its definition in `components/securitySchemes` is preserved.
  - After any edit or delete, the panel list updates immediately.
