# WP-056: Create reusable schema from inline response structure

- **ID**: WP-056
- **Description**: As a technical user, I want to create a reusable schema from an inline response structure, so I can standardize response models across endpoints.
- **Acceptance criteria**:
  - When a response media type uses an editable inline object schema, the UI exposes an action to create a schema from that inline structure.
  - Triggering the action asks for a target schema name before applying changes.
  - If the schema name is valid and unique, the app creates the schema in `components.schemas` from the current inline response structure.
  - After successful creation, the selected response schema is replaced with a `$ref` to the new schema.
  - Response metadata unrelated to schema shape (for example status code, media type, response description, and media-entry extras) remains unchanged.
  - If the user cancels, no schema is created and the response remains inline.
