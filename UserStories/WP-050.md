# WP-050: Apply alphabetical ordering for paths and schemas across the editor

- **ID**: WP-050
- **Description**: As a technical user, I want API paths and schemas to be ordered alphabetically in all relevant views, so I can find entries faster and get a consistent browsing experience.
- **Acceptance criteria**:
  - API paths are displayed in alphabetical order in every path list view, not only in the sidebar.
  - Schemas are displayed in alphabetical order in every schema list/select view, not only in the sidebar.
  - Sorting is case-insensitive and stable for names that differ only by casing.
  - When a path or schema is created, renamed, imported, or deleted, ordering updates automatically.
  - Sorting affects display order only and does not mutate OpenAPI object semantics.
  - Filtering continues to work correctly with sorted data.
