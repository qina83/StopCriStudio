# WP-002: User creates new OpenAPI specification

- **ID**: WP-002
- **Description**: As a technical user, I want to create a new OpenAPI specification by clicking a "Create" button, so I can start documenting my API from scratch with a structured editor interface.
- **Acceptance criteria**:
  - "Create" button is clearly visible on Welcome page
  - Clicking "Create" navigates to the specification editor interface
  - Editor displays a three-panel layout: left sidebar, header, and main content panel
  - Left sidebar contains navigation items:
    - Info (selectable item)
    - Path (accordion with + button to add paths)
    - Schema (accordion with + button to add schemas)
  - Info panel is selected by default upon creation
  - OpenAPI version is fixed at 3.0 (no version selection required)
  - Info panel displays input fields for:
    - Specification name (editable text field)
    - Specification version (editable text field)
    - OpenAPI version displays as 3.0 (read-only)
  - When user modifies specification info (name or version), the header displays the updated information in real-time
  - Changes are automatically saved to local storage (autosave - no manual save button needed)
  - New specification is created with default/minimal required fields
  - Create action does not interfere with existing specifications in local storage
