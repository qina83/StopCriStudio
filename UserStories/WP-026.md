# WP-026: Export request body to YAML

- **ID**: WP-026
- **Description**: As a technical user, I want the request body to be included in the exported OpenAPI YAML, so that the generated specification is complete and valid for consumers and tooling.
- **Acceptance criteria**:
  - The exported YAML includes a `requestBody` key under the correct operation (PUT, POST, or PATCH) when a body has been defined.
  - The `description` field is included in the export only when a non-empty value has been set.
  - The `required` field is always exported as an explicit boolean.
  - The `content` key contains exactly one media type entry matching the selected media type.
  - The schema under the media type is exported as an inline object (`type: object`) with a `properties` map reflecting the defined parameters.
  - Required properties are listed in the `required` array of the schema object, not on individual property nodes.
  - Nested objects export as nested `type: object` schemas with their own `properties` maps.
  - Array properties export with `type: array` and an `items` definition matching the configured item type.
  - Property descriptions, when set, are included as `description` fields on the respective schema property.
  - Operations with no body defined do not include a `requestBody` key in the export.
  - Non-eligible methods (GET, DELETE, etc.) never include a `requestBody` key in the export.
  - Generated YAML is valid for OpenAPI 3.0 and 3.1.

## User interaction flow

*The user finishes defining a POST body with media type `application/json` and two required properties. On export, the YAML contains a well-formed `requestBody` block under the POST operation with the correct schema.*
