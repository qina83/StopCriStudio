# WP-057: Validate and safely persist schema extraction from inline models

- **ID**: WP-057
- **Description**: As a technical user, I want safe validation and persistence when converting inline models to reusable schemas, so I can trust that my specification stays consistent.
- **Acceptance criteria**:
  - Schema creation from inline request body or response is blocked when the target schema name is empty, invalid, or already used.
  - Validation errors are shown with clear guidance, and no partial write is persisted when validation fails.
  - If creation succeeds, both changes are saved together: the new `components.schemas` entry and the `$ref` replacement at the original inline location.
  - If the original inline model is not visually editable (unsupported structure), the create-schema action is not available from that location.
  - Existing nested `$ref` values inside the extracted model are preserved as-is in the generated schema.
  - After successful extraction, the new schema is immediately available in schema selectors without requiring a page reload.
