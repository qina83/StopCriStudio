# WP-002.1: User manages path form visibility

- **ID**: WP-002.1
- **Description**: As a technical user, I want the path list and form to be toggled intelligently based on my actions, so I can have a focused editing experience when creating new paths and a clear view of existing paths when needed.
- **Acceptance criteria**:
  - When user clicks "Add Path" button in the Paths sidebar, the new path form appears in the main panel
  - The existing paths list is hidden when the new path form is displayed
  - Form displays alone in the main panel without list of existing paths below it
  - When user clicks on the "Path" sidebar item (accordion header), the new path form is hidden
  - The existing paths list appears in the main panel when the form is hidden
  - Clicking "Path" sidebar item toggles between form view and list view
  - Only one state is visible at a time: either the form OR the list, never both
  - List displays all previously created paths in a scrollable view
  - When user creates a new path and confirms, the form closes automatically and the updated list displays
  - The sidebar Paths accordion remains expanded when switching between form and list views
  - User can switch between adding a new path and viewing the list multiple times without issues
  - Form closure does not interfere with existing path data in local storage
